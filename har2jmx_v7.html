How to capture from your browser

  1. Open your banking app in Chrome / Edge / Firefox
  2. Press F12 → go to the Network tab
  3. Tick Preserve log, then click the clear (⊘) button
  4. Perform your business flow (login, view balance, transfer, etc.)
  5. Right-click anywhere in the request list → Save all as HAR with content
  6. Drop the saved .har file below — everything stays in your browser, nothing is uploaded

After upload, only your business calls are auto-selected. Static assets, analytics, OPTIONS preflights, heartbeats and failed (4xx/5xx) calls are auto-skipped — but listed for review. You can override anything per-row.

Authentication none ▾

How should the test plan handle auth tokens? Tokens often expire mid-test; the right strategy depends on your app.

For runtime override at run time, pass -JauthToken=... to JMeter on the command line.

The login request will be moved into a Once Only Controller (executes once per thread iteration). The token extracted from its response will be used by all subsequent requests in the iteration.

A Result Status Action Handler + If Controller is added: when any request returns 401/403, the login is re-executed and the token refreshed. Subsequent requests use the new token.

Default behavior: tokens detected in JSON responses or Set-Cookie headers are extracted via JSONPostProcessor / RegexExtractor and substituted as ${tokenName} in downstream requests. Works fine for short flows where token doesn't expire mid-test.
